# Code Swapper: AI-Driven Precision Patching Protocol

### 项目定义
这是一个基于 **LOCATE/REPLACE** 逻辑的代码修改解析工具。它旨在解决 AI 在处理 Python、AutoHotkey (AHK) 等缩进敏感型脚本时，因全量输出代码导致的 Token 浪费、推理延迟以及缩进崩溃问题。
同时，本项目极大降低了非专业开发者的协作门槛： 即使不具备深厚的编程背景，用户也能通过本工具精准执行 AI 给出的修改指令，彻底解决因手动粘贴导致的“找不到修改位置”或“缩进/格式错误”等损耗精力的问题。

### 核心亮点 (区别于传统 IDE 或 AI 助手)
* **确定性控制**：不同于 Cursor 或 Aider 的模糊 Diff，本项目强制执行“字节级锚点匹配”。如果 `LOCATE` 块与本地文件存在一个空格或缩进的差异，脚本将拒绝修改。这种“宁死不动”的机制确保了代码修改的绝对安全性。
* **算力极简主义**：在实际测试中，修改一个 1000 行的文件，本方案仅需输出不到 5% 的 Token。这种极高的能效比能显著降低大模型的推理支出。
* **逻辑审计优先**：工作流要求“先确认逻辑，后下发代码”。这种机制强制用户在补丁生成前进行逻辑审计，极大地降低了 AI 幻觉引发的生产事故。

### 工作流逻辑
1.  **逻辑确认**：用户与 AI 确定修改逻辑，达成一致。
2.  **补丁生成**：AI 按协议输出包含 `LOCATE`（定位块）和 `REPLACE`（替换块）的特定格式。
3.  **本地执行**：本脚本解析文本，定位锚点并完成物理替换。

### 适用场景与局限
* **适用**：对缩进有极致要求的环境（Python, AHK）；长脚本的频繁微调；追求极致 Token 经济性的开发者。
* **局限（重要）**：由于采用字节级匹配，**若用户在 AI 对话期间手动改动了本地代码格式**，会导致匹配失效。这被视为一种安全熔断，此时需要用户自行同步代码状态或切换至全量覆盖模式。
